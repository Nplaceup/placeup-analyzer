Nplaceup · Python 분석 데이터 스펙 명세서

Backend(Spring) ↔ Python 통신은 Redis 전용. Python이 산출하는 전체 데이터와 현재 payload 현황을 정의.

Redisanalysis:queue

Pythonbrpop → 분석 실행
→
Redisanalysis:result:queue

Springbrpop → DB 저장
단계큐 키방향전달값 요약
1차 요청 analysis:queue Spring → Python {"place_id": 123, "round": 1}
1차 결과 analysis:result:queue Python → Spring {"place_id": 123, "round": 1, "keywords": ["돌솥밥", ...]}
2차 요청 analysis:queue Spring → Python {"place_id": 123, "round": 2} (RankSearch 완료 후)
2차 결과 analysis:result:queue Python → Spring {"place_id": 123, "round": 2, "keywords": [{...}]}
💡 Python은 Redis 요청 수신 후 DB(RDS)에서 데이터를 직접 조회해 분석. Spring이 payload로 데이터를 전달하지 않음.
프론트가 필요한 모든 데이터는 round 2 result payload에 포함되어야 함 — Python 외부로 나갈 수 있는 유일한 경로가 Redis이기 때문.

seoScore 필드 상세 미전달 — 추가 가능

Python STAGE 6에서 이미 계산 완료. main.py STAGE 7 payload에 추가하면 즉시 전달 가능.

필드타입설명
total integer 최종 SEO 점수. 0~100 정수.
grade string 등급 문자열. 🟢 우수(80+) / 🟡 보통(60+) / 🟠 미흡(40+) / 🔴 취약
breakdown.keywordOptimization float 검색용 키워드 비율 + 카테고리 다양성. 최대 40점.
breakdown.reviewQuality float 평균 키워드 점수 + 일관성. 최대 30점.
breakdown.searchExposure float 상위 10위 키워드 수 + 기회 키워드 수. 최대 20점.
breakdown.competition float 경쟁도 낮은 키워드 비율. 최대 10점.
competitorAnalysis 필드 상세 미전달 — 추가 가능

동일 카테고리+지역 경쟁업체(리뷰수 상위 최대 10개) 기반 분석. 현재 블렌딩 내부에서만 소비됨.

필드타입설명
competitorCount integer 실제 분석된 경쟁업체 수. 최대 10.
advantageKeywords string[] 내 키워드 중 경쟁업체 전체에 없는 독점 키워드. 강점 표시용.
categoryGap object 카테고리별 내 키워드 수 vs 경쟁업체 평균 비교. 레이더 차트 등에 활용.
키: "음식" / "서비스" / "분위기" / "장소"
⚠ gap_keywords, rank_gap_keywords는 키워드 점수 계산에 내부적으로 이미 반영됨. 별도로 payload에 포함할 경우 BE가 이중 처리하지 않도록 주의.
