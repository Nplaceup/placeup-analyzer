백엔드 연동 스키마 명세서

백엔드(Java) ↔ Redis ↔ Python 분석 모듈 간 메시지 큐 기반 연동 명세.
Python 모듈은 REST API 없이 Redis Worker로 동작하며, 요청/결과를 모두 큐로 처리한다.

통신 방식
Redis 메시지 큐 (brpop / lpush)
요청 큐
analysis:request:queue
결과 큐
analysis:result:{requestId}
Java API 엔드포인트
GET /v1/openapi/place/recommend
Python 역할
Redis Worker (brpop 대기)
Java 결과 저장
place_recommended_keywords

⑦ Python → Redis 결과 페이로드 analysis:result:{requestId}
필드타입설명
requestId string 요청 시 전달받은 requestId 그대로 반환. Java에서 요청-결과 매핑에 사용
status string success error
recommendedKeywords array<KeywordItem> 추천 키워드 목록. score 내림차순 정렬
errorMessage string | null status=error 시 오류 메시지. 정상 시 null
recommendedKeywords 배열 원소 (KeywordItem)
필드타입설명
keywordstring추천 키워드 텍스트
monthlySearchVolumeinteger월간 검색량. 0 = 데이터 없음
rankNointeger | null현재 네이버 플레이스 순위. null = 미순위
competitionLevelstring높음 ≥10,000 / 중간 ≥1,000 / 낮음 <1,000
keywordPurposestringsearch SEO 최적화 대상 / marketing 광고·콘텐츠 소재
categorystring음식 장소 서비스 분위기 미분류
isOpportunityboolean기회 키워드 여부. 검색량 ≥1,000 이고 순위 10위 밖
scorefloatNLP 분석 최종 점수 (0~1)

Python 측 구현 필요 사항

현재 상태 vs 필요 변경

항목현재필요 변경
실행 방식 python main.py 직접 실행 Redis brpop 대기 루프 Worker로 변경
입력 데이터 DB에서 직접 조회 (place_id 하드코딩) Redis 페이로드에서 placeDetail, rankSearch 수신
결과 출력 콘솔 출력 + 로컬 DB upsert만 로컬 DB upsert 후 결과를 Redis lpush로 전송 추가
필드명 컨벤션 snake_case (monthly_search_volume) 결과 전송 시 camelCase 변환 필요 (monthlySearchVolume)
참고: Python Worker 구현 시 redis_worker.py 별도 파일로 분리 권장. main.py의 run(place_id) 함수를 그대로 호출하고, Worker는 Redis I/O만 담당하는 구조로 최소 변경 가능.
