recommend_keywords 테이블 명세서

NLP 키워드 분석 파이프라인의 최종 결과를 저장하는 로컬 PostgreSQL 테이블.
매장(place_id)별로 추천 키워드와 점수·메타데이터를 upsert 방식으로 관리한다.

테이블명
recommend_keywords
DB
Local PostgreSQL (WRITE_DB)
기본키
id (SERIAL)
유니크 제약
(place_id, keyword)
쓰기 방식
ON CONFLICT DO UPDATE (upsert)
파이프라인 위치
STAGE 5 (최종 저장)
데이터 생성 흐름
STAGE 0 DB 조회 (리뷰·매장정보) → 모듈1 기본 키워드 생성 → 모듈2 NLP (TF-IDF · CASE A/B/C · 스코어링) → 모듈3 경쟁업체 분석
                               → 블렌딩 가중치 합산 → STAGE 4 유도어 결합 → STAGE 5 recommend_keywords upsert
컬럼 명세
컬럼명 타입 NULL 기본값 설명
식별자
PKid SERIAL NOT NULL auto 자동 증가 기본키
FKplace_id INT NOT NULL — 매장 ID. RDS place_reviews.places_id 참조
UKkeyword VARCHAR(100) NOT NULL — 키워드 텍스트. 단어(unigram) 또는 복합어(bigram). 유도어 결합형 포함 (예: "파스타 맛집"). (place_id, keyword) 조합으로 유니크
종합 점수
score FLOAT NOT NULL — 블렌딩 최종 점수 (0~1). 모듈1·2·3 가중치 합산 후 정규화. 높을수록 추천 우선순위 높음
NLP 지표별 점수 (STAGE 3 · NLP 키워드만 유효, 나머지는 기본값)
tfidf_score FLOAT NOT NULL 0.0 TF-IDF 정규화 점수 (0~1). 리뷰 내 키워드 중요도. NLP 키워드만 유효값, 기본·경쟁업체 키워드는 0.0
sentiment_score FLOAT NOT NULL 1.0 감성 점수 (0~1). −1~1 범위를 0~1로 정규화. 현재 감성 사전 미연동으로 전체 1.0 고정 (추후 업데이트 예정)
recency_score FLOAT NOT NULL 0.0 최신성 점수 (0~1). 최근 리뷰에서 언급될수록 높음. 감쇠 공식: 1 / (1 + 경과개월 × 0.1)
consistency_score FLOAT NOT NULL 0.0 일관성 점수 (0~1). 여러 리뷰어가 독립적으로 언급할수록 높음. 언급 리뷰 수 / 전체 리뷰 수
키워드 분류
is_ngram BOOLEAN NOT NULL FALSE 복합어(bigram) 여부. 공백 포함 시 TRUE (예: "파스타 맛집" → TRUE, "파스타" → FALSE)
is_induced BOOLEAN NOT NULL FALSE 유도어 결합형 여부. STAGE 4에서 유도어 사전 기반으로 결합 생성된 키워드 (예: "파스타" → "파스타 맛집")
keyword_purpose VARCHAR(20) NOT NULL 'marketing' 키워드 용도. search 네이버 검색 노출 최적화 대상 키워드 / marketing 광고·콘텐츠 소재용 키워드
category VARCHAR(20) NOT NULL '미분류' 의미 카테고리. 음식 장소 서비스 분위기 미분류
순위 · 검색량 메타데이터 (STAGE 2.5 / RDS 크롤링 데이터 기반)
case_type CHAR(1) NOT NULL 'C' 키워드 출처 분류. A NLP 추출 ∩ 네이버 순위 데이터 동시 존재 / B 순위 데이터만 존재 (리뷰 미언급) / C NLP 추출만 존재 (순위 없음)
rank_no INT NULL 가능 NULL 현재 네이버 플레이스 검색 순위. 1이 최상위. NULL = 해당 키워드 순위 데이터 없음 (CASE C)
rank_no_change INT NOT NULL 0 전일 대비 순위 변동. 양수 = 상승, 음수 = 하락, 0 = 유지
monthly_search_volume INT NOT NULL 0 월간 네이버 검색량. 0 = 검색량 데이터 없거나 미집계 키워드
mention_count INT NOT NULL 0 리뷰 언급 건수 (해당 키워드가 등장한 리뷰 수). CASE B(순위only) 키워드는 0
competition_level VARCHAR(10) NOT NULL '낮음' 월간 검색량 기반 경쟁 강도. 높음 ≥ 10,000 / 중간 ≥ 1,000 / 낮음 < 1,000
is_opportunity BOOLEAN NOT NULL FALSE 기회 키워드 여부. monthly_search_volume ≥ 1,000 AND rank_no > 10 또는 NULL 인 경우 TRUE. 검색 수요는 있으나 현재 노출이 부족한 키워드
타임스탬프
analyzed_at TIMESTAMP NOT NULL NOW() 파이프라인 분석 실행 시각. upsert 시 매번 갱신되므로 마지막 분석 시점 추적 가능
제약 조건 및 인덱스

제약 조건

PRIMARY KEY (id)
UNIQUE (place_id, keyword) — upsert 충돌 기준키. 동일 매장·키워드 조합 중복 불가

권장 인덱스 (미적용 시 쿼리 성능 저하 가능)

idx_rk_place_score ON recommend_keywords (place_id, score DESC) — 매장별 상위 키워드 조회
idx_rk_opportunity ON recommend_keywords (place_id, is_opportunity) — 기회 키워드 필터
idx_rk_analyzed_at ON recommend_keywords (analyzed_at DESC) — 최신 분석 결과 조회
